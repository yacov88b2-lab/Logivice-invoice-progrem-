# רשימת בדיקות לפני Release — ראשון 17/05/2026
**מבצע הבדיקות:** תומר  
**סביבה:** Staging  
**יעד:** לאשר שהמערכת עובדת מקצה לקצה לפני העלאה לProduction

---

## 🔴 חלק א' — לקוח חדש (בדיקה קריטית)
> זהו התרחיש המרכזי שהמערכת נבנתה עבורו. חייב לעבור לפני Release.

### 1. העלאת תבנית חדשה
- [ ] היכנס לממשק Admin
- [ ] לחץ "Upload Pricelist"
- [ ] בחר קובץ Excel של לקוח שעדיין **אין** לו תבנית במערכת
- [ ] וודא שהתבנית מופיעה ברשימה לאחר ההעלאה
- [ ] לחץ על התבנית — וודא שהמבנה שלה נקרא נכון (שמות Sheets, מספר שורות)

### 2. בניית חוק מהממשק
- [ ] היכנס ל-Rule Control
- [ ] בחר את הלקוח החדש
- [ ] לחץ "New Rule"
- [ ] עבור דרך האשף — תאר בשפה חופשית מה ההתאמה צריכה לעשות
- [ ] וודא שה-AI מציע שלבים הגיוניים (אם יש קרדיטים) — אחרת בנה ידנית
- [ ] שמור את החוק

### 3. זרימת אישור החוק
- [ ] הרץ **Test** על החוק עם נתוני דוגמה
- [ ] וודא שהסטטוס משתנה ל **"Tested"** לאחר בדיקה מוצלחת
- [ ] לחץ **Approve** — וודא שהסטטוס משתנה ל **"Approved"**
- [ ] לחץ **Enable** — וודא שהחוק פעיל
- [ ] נסה לערוך את החוק הפעיל — צריך להיחסם עם הודעה ברורה

### 4. הפקת חשבונית עם החוק הפעיל
- [ ] עבור לממשק המשתמש (User Dashboard)
- [ ] בחר את הלקוח החדש ואת התבנית שהעלית
- [ ] בחר טווח תאריכים
- [ ] לחץ **Preview**
- [ ] וודא שמוצג **"Active rule: [שם החוק]"** ולא "legacy mapper only"
- [ ] בדוק את אחוז ההתאמה — האם הגיוני?
- [ ] לחץ **Generate Invoice**
- [ ] הורד את הקובץ ופתח ב-Excel
- [ ] **⚠️ חשוב:** וודא שהכמויות מולאו בשורות הנכונות בתבנית

## 🟡 חלק ב' — תור הסקירה (Review Queue)
> לבצע עם לקוח שיש לו עסקאות עם התאמות מרובות (Sensos הוא דוגמה טובה)

### 5. תצוגת הסקירה
- [ ] הרץ Preview עבור Sensos
- [ ] וודא שמספר ההזמנה (Order Number) מוצג **גדול ובולט** בראש כל כרטיס
- [ ] וודא שיש **שני אזורים נפרדים**: "Confirm match" ו-"Choose the right row"
- [ ] בדוק שכל אפשרות מציגה: שם השורה, שם ה-Sheet, ומחיר ליחידה

### 6. פעולות בתור הסקירה
- [ ] בחר אפשרות באחד מהכרטיסים — וודא שנבחרת (מסומנת בכחול)
- [ ] לחץ **"Flag for later"** על כרטיס אחר — וודא שהוא עובר לתחתית
- [ ] פתח את רשימת "Flagged for later" — וודא שהכרטיס שם
- [ ] לחץ **Unflag** — וודא שהוא חוזר לרשימה הראשית
- [ ] אשר מספיק בחירות ולחץ **"Confirm & update preview"**
- [ ] וודא שהתצוגה מתעדכנת ואחוז ההתאמה עולה

## 🟢 חלק ג' — בדיקות כלליות

### 7. קובץ אקסל מוצא
- [ ] הורד חשבונית מוגמרת
- [ ] פתח ב-Excel — וודא שהצבעים והעיצוב של התבנית **נשמרו**
- [ ] וודא שאין Sheets ריקים או שבורים
- [ ] וודא שהכמויות מולאו (לא אפס בכל מקום)

### 8. מניעת כפילויות
- [ ] הרץ אותה חשבונית (אותו לקוח, אותו תאריך) **פעמיים**
- [ ] וודא שבפעם השנייה מוצגת אזהרה על כפילות

### 9. ממשק כללי
- [ ] הלוגו מוצג נכון בפינה השמאלית העליונה
- [ ] אין הודעות שגיאה אדומות בממשק
- [ ] לחץ כפתור "Report Bug" — וודא שהטופס נפתח ונשלח
